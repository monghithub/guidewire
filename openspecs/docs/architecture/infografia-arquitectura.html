Patrones de Comunicacion

Flujos de Integracion

Cuatro patrones principales: REST sincrono, puente SOAP, eventos asincrono y reglas de negocio

01

REST Sync REST Sincrono

Flujo request-response clasico. El cliente envia una peticion REST que atraviesa el API Gateway, llega al Camel Gateway y se enruta al servicio correspondiente.

Client → 3Scale APIcast → Camel Gateway → PolicyCenter Mock → JSON Response
POST /api/v1/policies Content-Type: application/json
02

SOAP Puente SOAP/REST

Guidewire envia mensajes SOAP/XML. Camel CXF los recibe, transforma XML a JSON mediante dataFormat=PAYLOAD, y enruta al backend REST correspondiente.

PolicyCenter → CXF Endpoint → XML/JSON Transform → REST Backend
POST /ws/policycenter Content-Type: text/xml
03

Async Eventos Asincrono (Kafka)

Flujo event-driven completo. Una factura llega por REST, Camel produce un evento Avro a Kafka, Billing Service consume, persiste en PostgreSQL y publica el cambio de estado.

POST /gw-invoices → Camel → InvoiceCreated → Kafka → Billing → PostgreSQL → StatusChanged
AVRO Schema: InvoiceCreated Apicurio Registry validated
04

Rules Reglas de Negocio (Drools)

Evaluacion de reglas en tiempo real. Un servicio envia datos al motor Drools, que ejecuta reglas DRL y devuelve un score de fraude o validacion de poliza.

POST /rules/fraud-check → Drools Engine → DRL Rules → Fraud Score
Response: { "fraudScore": 0.87, "riskLevel": "HIGH" }
